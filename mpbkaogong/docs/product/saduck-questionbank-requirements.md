# SaDuck 题库功能需求文档

分析对象：

- 历年试卷：https://saduck.top/questionBank/overTheYears.html
- 专项练习：https://saduck.top/questionBank/specialProject.html
- 练习记录：https://saduck.top/questionBank/practiceLog.html

## 1. 产品概述

SaDuck 题库是面向公考行测刷题的在线练习系统，核心目标是让用户完成三类学习任务：

- 按真实试卷刷题：历年试卷。
- 按知识点定向训练：专项练习、每日一练。
- 复盘历史表现：练习报告、错题再练、背题模式。

系统围绕同一个答题组件运转，试卷、专项、错题、历史记录都会进入统一的答题页。

## 2. 用户与权限

目标用户：

- 未登录用户：可访问题库页面，但无法正式进入练习。
- 已登录用户：可获取试卷、专项、记录、错题。
- 会员用户：可访问更多题库或相关会员内容。

权限规则：

- 拉取题目、专项树、练习记录、错题均需要 token。
- 未登录时提示“请先登录”，部分流程会跳转到个人中心。
- 历年试卷列表本身可加载，但进入试卷详情需要登录。
- 提交练习会写入练习记录。

## 3. 信息架构

题库包含三个主入口：

- 历年试卷：`/questionBank/overTheYears.html`
- 专项练习：`/questionBank/specialProject.html`
- 练习记录：`/questionBank/practiceLog.html`

共用答题组件支持：

- 单选、多选、判断题。
- 材料题展示。
- 计时。
- 草稿纸。
- 答题卡。
- 提交。
- 结果统计。
- 解析展示。
- 历史记录回看。
- 错题练习。
- 背题模式。

## 4. 历年试卷模块

目标：用户按考试年份或试卷分类选择完整试卷进行练习。

页面功能：

- 加载试卷分类列表。
- 顶部展示分类 tab，例如不同年份、地区、考试类型。
- 点击分类后展示该分类下的试卷列表。
- 每张试卷显示试卷名称和难度评分。
- 点击试卷进入答题页。
- 页面提供夸克网盘、百度网盘资料入口。

核心流程：

1. 用户进入历年试卷页。
2. 系统请求试卷分类列表。
3. 用户选择分类。
4. 用户点击某套试卷。
5. 系统校验登录。
6. 系统拉取试卷题目。
7. 进入统一答题页。

接口需求：

- `POST /api/tk/itemizes?type=1`
  - 获取历年试卷分类与试卷列表。
  - 返回内容为加密字符串，前端解密后使用。
- `POST /api/tk/sourceInfo`
  - 获取某套试卷详情。
  - 请求体包含加密后的 `sid`。
  - 需要 token。

## 5. 专项练习模块

目标：用户按知识点自由组题，进行定向训练。

页面功能：

- 展示专项分类树。
- 支持展开/收起节点。
- 支持选择知识点。
- 支持选择父节点时批量选择子节点。
- 支持给每个选中知识点设置题量。
- 已选知识点进入汇总区，并显示预计总题数。
- 支持清空选择。
- 支持开始练习。
- 支持每日一练。
- 支持难度筛选。

题量可选：

- 1 题
- 2 题
- 3 题
- 4 题
- 5 题
- 10 题
- 15 题
- 20 题

难度规则：

- 无：包含全部题目。
- 易：只包含简单题。
- 中：只包含一般题。
- 难：只包含困难题。

专项组卷规则：

- 必须选择至少一个叶子知识点。
- 预计总题数不能少于 5。
- “资料分析”“篇章阅读”“组合排列（材料）”等材料类专项不能和其他专项混合练习，只能单独练。
- 若接口返回 0 道题，提示“未查询到相关题目”。

每日一练：

- 点击首页 banner 触发。
- 系统直接请求每日练习题。
- 成功后进入统一答题页。

本地缓存：

- 保存用户上次选择的专项节点。
- 保存用户中途未完成的练习。
- 用户下次进入时询问是否继续上次练习。

接口需求：

- `POST /api/tk/problemTagNew?type=1`
  - 获取专项分类树。
  - 需要 token。
- `POST /api/tk/tagInfo`
  - 根据知识点、题量、难度生成专项练习。
  - 请求体示例：

```json
{
  "reqs": [
    { "name": "知识点ID", "num": 5 }
  ],
  "difficulty": null
}
```

- `POST /api/tk/getPractice`
  - 获取每日一练。
  - 需要 token。

## 6. 统一答题页

目标：承载所有题目练习场景。

适用来源：

- 历年试卷。
- 专项练习。
- 每日一练。
- 练习记录回看。
- 错题练习。
- 错题背题。

核心展示：

- 试卷/练习名称。
- 当前用时。
- 当前题号 / 总题数。
- 上一题、下一题。
- 题目所属知识点。
- 材料内容。
- 题干。
- 选项。
- 多选题提示。
- 答题卡。
- 草稿纸。
- 暂停按钮。
- 返回按钮。
- 提交按钮。

题型支持：

- 单选题：点击选项后记录答案，普通练习中可自动进入下一题。
- 多选题：支持勾选多个选项，并显示多选提示。
- 判断题：答案展示为“正确/错误”。

草稿纸功能：

- 支持开启/关闭草稿纸。
- 支持鼠标或触摸绘制。
- 支持清空草稿。
- 切题时关闭草稿状态。
- 草稿层覆盖在题目区域上方。

计时功能：

- 进入答题页后开始计时。
- 支持暂停。
- 暂停时弹窗显示当前计时。
- 点击“继续作答”恢复计时。
- 记录每道题答题用时。

答题卡功能：

- 按模块分组展示题号。
- 未提交时显示当前题、已答、未答。
- 提交后显示当前题、正确、错误、未答。
- 点击题号跳转到对应题目。

中途保存：

- 作答时自动把当前练习写入 `localStorage.lx_his`。
- 再次进入题库时询问是否继续。
- 提交或放弃后清除本地缓存。

提交规则：

- 用户点击提交后统计已答题数、正确题数、错误题数、未答题数。
- 记录正确题 ID、错误题 ID。
- 按 tag 聚合正确/错误题。
- 如果完成率大于 50%，弹窗提示剩余未答题数，并保存记录。
- 如果完成率不足 50%，提示“系统不会保存本次记录”，确认后只展示结果，不写练习记录。
- 未登录不能提交。

提交接口：

- `POST /api/tk/addRecord`
  - 写入练习记录。
  - 请求包含：
    - `tname`
    - `total`
    - `timeTaken`
    - `correct`
    - `wrong`
    - `unanswered`
    - `json`
    - `correctIds`
    - `errorIds`
    - `tkWrongAnswerIds`

提交后展示：

- 正确率仪表盘。
- 正确数量。
- 错误数量。
- 未答数量。
- 每题正确答案。
- 用户答案。
- 全站正确率。
- 答题用时。
- 解析。
- 题目来源。

## 7. 背题模式

目标：用于错题复习或记忆训练。

行为差异：

- 题目仍使用统一答题组件。
- 用户选择答案后立即显示答案和解析。
- 答题卡状态显示正确/错误。
- 标题会追加“背题”。
- 适合快速浏览错题，而不是完整模拟练习。

## 8. 练习记录模块

目标：用户查看历史练习表现和复盘。

页面分为两个 tab：

- 练习报告。
- 我的错题。

练习报告功能：

- 展示整体统计：
  - 整体正确率。
  - 练习次数。
  - 练习题数。
  - 答对题数。
  - 累计用时。
- 展示模块统计：
  - 各模块正确率。
  - 子模块正确率。
- 展示历史练习列表：
  - 练习名称。
  - 练习时间。
  - 答题用时。
  - 正确/错误/未答比例。
- 支持分页。
- 点击某条练习记录可进入历史回看。

历史回看：

- 请求历史记录详情。
- 解析保存的题目 JSON。
- 进入答题页。
- `isHis=true`，默认视为已提交状态。
- 可查看答案、解析、作答情况。

接口需求：

- `POST /api/tk/recordNew?type=1`
  - 获取练习记录与模块统计。
- `POST /api/tk/recordInfo?id={recordId}`
  - 获取单次练习详情。

## 9. 我的错题模块

目标：用户按错题知识点复习。

页面功能：

- 获取错题分类。
- 每个分类显示 tag 和错题列表。
- 支持展开/收起分类。
- 错题内容会过滤图片，只展示文本内容。
- 每个错题分类提供两个动作：
  - 背题。
  - 练习。

错题练习规则：

- 某分类错题必须大于 10 道，否则提示“错题太少了，攒多一点再来吧！”。
- 系统从该分类错题中随机抽取 10 道。
- 请求错题详情后进入答题页。
- 可选择普通练习或背题模式。

接口需求：

- `POST /api/tk/getCt`
  - 获取错题分类和错题列表。
- `POST /api/tk/getCtInfo`
  - 获取错题练习详情。
  - 请求体示例：

```json
{
  "indexs": [123, 456, 789],
  "name": "错题分类名"
}
```

## 10. 数据对象需求

题目对象核心字段：

```json
{
  "id": 123,
  "title": "题干 HTML",
  "material": "材料 HTML",
  "options": "A选项#B选项#C选项#D选项",
  "type": "single",
  "correctAnswer": "0",
  "analysis": "解析 HTML",
  "globalAccuracy": "全站正确率",
  "source": "题目来源",
  "tag": "知识点",
  "userAnswer": null,
  "memorize": null,
  "timeSpent": 0
}
```

试卷模型 `model`：

- 用于答题卡分组。
- 每组包含：
  - `name`
  - `snum`
  - `enum`

练习记录对象：

```json
{
  "id": 1,
  "tname": "练习名称",
  "total": 100,
  "timeTaken": 3600,
  "correct": 80,
  "wrong": 15,
  "unanswered": 5
}
```

模块统计对象：

```json
{
  "name": "模块名",
  "zqc": 10,
  "cwc": 3,
  "children": []
}
```

## 11. 异常与提示

通用异常：

- 未登录：提示“请先登录”，部分场景跳转个人中心。
- token 失效：接口返回 100 或 101 时提示重新登录。
- 题目为空：提示“未查询到相关题目”。
- 错题不足：提示“错题太少了，攒多一点再来吧！”。
- 未提交离开：提示“尚未提交，确定离开本页面吗？”。
- 完成率不足：提示不会保存本次记录。

## 12. 可学习的产品设计点

- 一个统一答题组件承载多种来源，复用度高。
- 历年试卷、专项练习、错题练习都能沉淀到练习记录。
- 通过 `localStorage` 做低成本中途恢复。
- 提交后才展示答案和解析，适合模拟练习。
- 背题模式立即展示解析，适合复习。
- 错题系统不是简单列表，而是支持随机抽题再练。
- 答题卡按模块分组，对行测这种多模块试卷很合适。
- 专项练习允许用户控制知识点和题量，但对材料题做了混练限制。
- 完成率低于 50% 不保存记录，避免污染学习统计。

## 13. MVP 复刻范围建议

第一版：

- 题库首页。
- 历年试卷列表。
- 试卷详情拉题。
- 统一答题页。
- 提交后展示正确率、答案、解析。
- 本地或后端保存练习记录。

第二版：

- 专项分类树。
- 按知识点组题。
- 难度筛选。
- 每日一练。

第三版：

- 练习报告。
- 模块正确率。
- 错题本。
- 错题练习。
- 背题模式。
- 草稿纸与单题用时统计。
