# 公考题库系统页面与交互设计文档

适用范围：

- Next.js Web/PWA 第一版。
- 移动端优先设计。
- 后期 React Native 可复用交互模型。

关联文档：

- `saduck-questionbank-requirements.md`
- `saduck-technical-architecture.md`
- `saduck-database-design.md`
- `saduck-api-design.md`

## 1. 设计目标

本系统的核心使用场景是移动端刷题，因此交互设计优先保证：

- 单手可操作。
- 阅读题干和材料不被控件干扰。
- 选项点击区域足够大。
- 答题卡、提交、暂停、返回始终容易触达。
- 刷题中断后可以恢复。
- 提交前不给答案，提交后复盘清晰。
- 背题模式立即反馈，适合复习。

第一版不做营销首页，登录后直接进入题库工作台。

## 2. 信息架构

主导航：

```txt
题库
├─ 历年试卷
├─ 专项练习
├─ 每日一练
├─ 练习记录
└─ 我的错题

我的
├─ 学习统计
├─ 会员状态
└─ 设置
```

移动端底部 Tab：

- 题库
- 练习
- 记录
- 我的

桌面端侧边导航：

- 使用左侧固定侧栏。
- 当前模块高亮。
- 内容区最大宽度限制，避免长行阅读疲劳。

## 3. 全局布局

### 3.1 移动端布局

视口宽度：`< 768px`

结构：

```txt
┌──────────────────────┐
│ 顶部栏：标题 / 返回 / 更多 │
├──────────────────────┤
│ 页面主体，可滚动          │
│                      │
├──────────────────────┤
│ 底部 Tab 或操作栏        │
└──────────────────────┘
```

规则：

- 底部固定区域需要适配 `safe-area-inset-bottom`。
- 页面主体底部预留操作栏高度，避免内容被遮挡。
- 题目选项按钮最小高度 48px。
- 主要按钮宽度尽量铺满。
- 弹窗优先用底部抽屉，危险确认使用居中 AlertDialog。

### 3.2 桌面端布局

视口宽度：`>= 1024px`

结构：

```txt
┌──────────┬────────────────────────┐
│ 侧边导航  │ 页面主体                 │
│          │                        │
│          │ 右侧可固定答题卡/统计面板  │
└──────────┴────────────────────────┘
```

规则：

- 列表页内容宽度不超过 1200px。
- 答题页使用题目区 + 答题卡双栏。
- 材料题可使用左右分栏。

## 4. 全局交互规范

### 4.1 Loading

使用场景：

- 首次加载列表。
- 创建练习。
- 提交练习。
- 拉取历史记录详情。

表现：

- 列表页使用骨架屏。
- 按钮触发的动作使用按钮 loading。
- 提交练习使用全屏 loading，避免重复提交。

### 4.2 Toast

用于轻量反馈：

- 保存成功。
- 登录失效。
- 未选择知识点。
- 题目为空。
- 错题不足。

位置：

- 移动端：顶部居中或底部操作栏上方。
- 桌面端：右上角。

### 4.3 Dialog

用于强确认：

- 未提交离开。
- 提交练习。
- 完成率不足 50%。
- 放弃上次练习。

按钮顺序：

- 主操作在右侧。
- 取消在左侧。
- 危险操作使用红色按钮。

### 4.4 Empty State

空态必须给出下一步动作。

示例：

- 无练习记录：显示“还没有练习记录”，按钮“去刷一套试卷”。
- 无错题：显示“当前没有错题”，按钮“开始专项练习”。
- 无搜索结果：显示“没有找到匹配内容”，按钮“清空筛选”。

## 5. 登录与权限交互

### 5.1 未登录访问题库

用户可看到：

- 试卷列表。
- 基础说明。
- 登录入口。

用户不可做：

- 开始练习。
- 查看练习记录。
- 查看错题。
- 提交答案。

触发受限动作时：

```txt
Toast：请先登录
Action：跳转 /login?next=当前路径
```

### 5.2 会员限制

会员内容卡片显示：

- 标识：会员。
- 文案：开通会员后可练习。
- 按钮：查看会员权益。

点击会员试卷：

- 已登录非会员：打开会员权益页或弹窗。
- 未登录：先登录。

## 6. 题库首页

路径：

```txt
/question-bank
```

目标：

- 给用户一个开始刷题的入口。
- 聚合最近练习、每日一练、错题提醒。

移动端布局：

```txt
顶部：题库

今日卡片
└─ 每日一练 / 继续上次练习

快捷入口
└─ 历年试卷 / 专项练习 / 我的错题 / 练习记录

学习概览
└─ 正确率 / 今日题数 / 错题数
```

交互：

- 有未完成练习时，顶部优先显示“继续上次练习”。
- 没有未完成练习时，显示“每日一练”。
- 错题数大于 0 时显示错题提醒。

## 7. 历年试卷页

路径：

```txt
/question-bank/papers
```

目标：

- 用户按年份、地区、考试类型找到试卷并开始练习。

页面结构：

```txt
顶部栏：历年试卷
筛选区：年份 / 地区 / 类型
试卷列表
  试卷卡片
    标题
    年份 / 地区 / 类型
    题量
    难度
    会员标识
    开始按钮
```

移动端交互：

- 筛选器使用横向滚动 chips。
- 高级筛选使用底部抽屉。
- 点击试卷卡片进入详情或直接创建练习。
- 点击“开始”后按钮 loading。

桌面端交互：

- 筛选器平铺在列表上方。
- 试卷列表使用两列或三列卡片。

状态：

- Loading：试卷卡片骨架。
- Empty：无匹配试卷，提供清空筛选。
- Error：加载失败，提供重试。

## 8. 试卷详情页

路径：

```txt
/question-bank/papers/[paperId]
```

目标：

- 开始前展示试卷信息，让用户确认。

页面内容：

- 试卷标题。
- 年份、地区、类型。
- 题量。
- 预计用时。
- 难度。
- 模块分布。
- 开始练习按钮。

- 点击“开始练习”创建 `PracticeSession`。
- 创建成功后跳转 `/practice/[sessionId]`。
- 如果有同一试卷未完成练习，提示继续或重新开始。

## 9. 专项练习页

路径：

```txt
/question-bank/special
```

目标：

- 用户按知识点、题量、难度组卷。

页面结构：

```txt
顶部栏：专项练习
每日一练 Banner
难度选择：不限 / 易 / 中 / 难
专项分类树
已选汇总
底部操作栏：清空 / 开始练习
```

移动端交互：

- 分类树纵向展示。
- 父节点点击展开/收起。
- checkbox 选择节点。
- 题量选择使用 stepper 或 select。
- 已选汇总默认折叠，点击后用底部抽屉展示。
- 底部固定操作栏显示：
  - 已选 N 类
  - 预计 M 题
  - 开始练习

专项树节点：

```txt
[ ] 资料分析        5题  >
    [ ] 增长率      5题
    [ ] 比重        5题
```

难度选择：

- 使用 segmented control。
- 默认“不限”。

业务校验：

- 未选知识点时，开始按钮 disabled。
- 总题数少于 5 时，点击提示“预估总题数太少了”。
- 材料类专项混选时，提示“材料类专项只能单独练习”。

## 10. 每日一练入口

入口位置：

- 题库首页顶部卡片。
- 专项练习页顶部 Banner。

卡片内容：

- 标题：每日一练。
- 副标题：不积跬步，无以至千里。
- 按钮：开始今日练习。

交互：

- 点击后创建每日一练 session。
- 已完成今日练习时，按钮改为“查看今日记录”。

## 11. 答题页

路径：

```txt
/practice/[sessionId]
```

目标：

- 用户完成一次练习或复盘。

### 11.1 答题页整体布局

移动端：

```txt
┌────────────────────────┐
│ 顶部栏：返回 / 标题 / 时间 │
├────────────────────────┤
│ 题号：12 / 120          │
│ 标签：资料分析           │
│ 材料折叠区               │
│ 题干                    │
│ 选项 A                  │
│ 选项 B                  │
│ 选项 C                  │
│ 选项 D                  │
│ 解析区（提交后显示）       │
├────────────────────────┤
│ 底部操作栏：上一题 答题卡 下一题 │
└────────────────────────┘
```

桌面端：

```txt
┌───────────────────┬──────────────┐
│ 题目区             │ 答题卡         │
│ 材料 / 题干 / 选项  │ 提交 / 暂停    │
│ 解析               │ 统计           │
└───────────────────┴──────────────┘
```

### 11.2 顶部栏

内容：

- 返回按钮。
- 练习标题。
- 计时器。

移动端标题：

- 超长省略。
- 点击标题可打开完整标题 tooltip 或 sheet。

计时器：

- 格式：`HH:mm:ss`。
- 暂停后停止增长。

### 11.3 题干与材料

材料题：

- 默认展开前 160px 高度。
- 点击“展开材料”查看全文。
- 长材料使用独立滚动区域。

普通题：

- 不显示材料区。

题干：

- 支持富文本。
- 图片最大宽度 100%。
- 表格可横向滚动。

### 11.4 选项

单选：

- 点击选项即选中。
- 普通练习可自动跳到下一题。
- 用户可返回修改，直到提交。

多选：

- 使用 checkbox。
- 不自动跳题。
- 显示提示：本题为多选题，请选择所有正确答案。

判断：

- 使用两个大按钮：正确 / 错误。

选项状态：

| 状态 | 表现 |
|---|---|
| 默认 | 白底边框 |
| 已选 | 主色边框 + 浅色背景 |
| 正确 | 绿色 |
| 错误 | 红色 |
| 禁用 | 透明度降低 |

### 11.5 底部操作栏

移动端固定底部：

```txt
[上一题] [答题卡] [下一题]
```

补充操作：

- 暂停。
- 草稿。
- 提交。

这些操作放在“更多”菜单或答题卡抽屉顶部。

桌面端：

- 上一题/下一题在题目上方。
- 提交/暂停/返回在右侧答题卡中。

### 11.6 答题卡

移动端：

- 使用底部抽屉。
- 点击“答题卡”打开。
- 抽屉高度 70vh。

内容：

- 提交按钮。
- 暂停按钮。
- 返回按钮。
- 分模块题号。
- 状态图例。

未提交状态：

- 当前：蓝色。
- 已答：绿色。
- 未答：灰色。

提交后状态：

- 当前：蓝色。
- 正确：绿色。
- 错误：红色。
- 未答：灰色。

### 11.7 草稿纸

入口：

- 答题页右下角悬浮按钮或底部“更多”菜单。

移动端表现：

- 近全屏半透明浮层，保留题号、计时和底部操作可见。
- 题目区域右下角轻量工具条：画笔 / 橡皮 / 撤销 / 清空 / 关闭。
- 支持手指绘制。

桌面端表现：

- 题目区域浮层。
- 支持鼠标绘制。

规则：

- 切换题目时关闭草稿纸。
- 草稿内容按题保存到本地 IndexedDB。
- 提交后结果页和错题回看中，草稿纸以只读方式展示。
- 草稿纸开启时选项不可点击，避免误触。

### 11.8 暂停

触发：

- 点击暂停按钮。

表现：

- 计时停止。
- 显示居中弹窗：
  - 标题：作答已暂停。
  - 当前用时。
  - 按钮：继续作答。

规则：

- 暂停期间不可操作题目。
- 返回或关闭页面仍触发未提交确认。

### 11.9 提交

触发：

- 点击答题卡中的提交按钮。

确认弹窗：

```txt
您已作答 80 题，还有 40 题未作答，确定要提交吗？
[取消] [确定提交]
```

完成率不足 50%：

```txt
您本次完成率不足 50%，系统不会保存本次记录，确定要提交吗？
[继续作答] [仍然提交]
```

提交中：

- 全屏 loading。
- 禁止重复点击。

提交成功：

- 留在当前答题页。
- 状态切换为结果模式。
- 显示正确率和答案解析。

提交失败：

- Toast：提交失败，请稍后重试。
- 保留本地答题状态。

## 12. 结果复盘页状态

提交后同一个答题页进入复盘状态。

显示内容：

- 正确率仪表。
- 正确数。
- 错误数。
- 未答数。
- 每题正确答案。
- 我的答案。
- 全站正确率。
- 本题用时。
- 解析。
- 题目来源。

交互：

- 答题卡点击跳转题目。
- 选项不可再修改。
- 可筛选：
  - 全部。
  - 只看错题。
  - 只看未答。
- 可加入收藏或标记掌握，第一版可后置。

## 13. 背题模式

适用来源：

- 我的错题。
- 未来收藏题。

行为：

- 用户选择答案后立即显示答案和解析。
- 不需要最终提交才看解析。
- 答题卡实时显示正确/错误。
- 标题显示“背题”标识。

移动端提示：

```txt
背题模式：选择后立即显示解析
```

## 16. 本地恢复交互

触发条件：

- 存在未提交的本地练习状态。
- 或服务端存在 `IN_PROGRESS` session。

弹窗：

```txt
是否继续上次练习？
《资料分析专项练习》
已用时 12:30，已答 8/20 题。
[放弃] [继续练习]
```

规则：

- 继续：恢复到上次题号。
- 放弃：清除本地缓存，可标记服务端 session 为 `ABANDONED`。
- 如果本地和服务端都有进度，优先使用更新时间较新的。

## 17. 离线与弱网交互

弱网策略：

- 当前练习已加载后，可继续本地作答。
- 顶部显示弱网提示：网络不稳定，答案已保存到本地。
- 提交失败时保留本地提交草稿。

离线状态：

- 列表页不可加载新数据。
- 答题页可继续当前 session。
- 提交按钮变为“网络恢复后提交”或提示稍后重试。

## 18. 可访问性与键盘操作

基础要求：

- 所有按钮有明确文本或 `aria-label`。
- 选项支持键盘 Tab 聚焦。
- 单选/多选语义正确。
- 答题卡题号按钮可聚焦。
- Toast 不承载唯一关键信息，重要错误同时在页面显示。

桌面端快捷键：

- `←` 上一题。
- `→` 下一题。
- `A/B/C/D` 选择选项。
- `Space` 打开/关闭答题卡。
- `S` 提交前聚焦提交按钮。

移动端不展示快捷键说明。

## 21. 验收标准

移动端答题页：

- iPhone SE 宽度下无横向滚动。
- 底部操作栏不遮挡选项。
- 选项点击区域高度不小于 48px。
- 答题卡抽屉能完整浏览所有题号。
- 长材料可独立滚动或折叠展开。
- 提交失败后答案不丢失。

桌面端答题页：

- 题目区和答题卡并排展示。
- 答题卡滚动不影响题目区阅读。
- 快捷键可用。

列表页：

- Loading、Empty、Error 三态齐全。
- 筛选后状态可恢复或反映在 URL query 中。

记录页：

- 点击历史记录可完整回看。
- 错题不足时有明确提示。

权限：

- 未登录访问受限动作会跳转登录。
- 非会员访问会员资源有明确提示。
